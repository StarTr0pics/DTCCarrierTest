    Changelog

    All notable changes to SDK will be documented in this file.

    [Unreleased]

    • Think about moving SDK to separate repository
    • Removed iGo references from SDK documentation

    [0.106.*]

    Added

    • methods to shared SDK:

      • needToAutoValidateScreens
      • autoValidateScreens

    • Runtime core API methods:

      • openNextVisibleScreen
      • openScreenByName
      • openFirstIncompleteScreen

    Changed

    • Runtime core API methods:
      • showLoader: added new param - loaderId, added ability to pass custom message which would be shown under spinner and updated docs
      • hideLoader: added new param - loaderId, update docs
      • renderUploadFileComponent method receives now new properties: maxFileNameLength, maxDescriptionLength; method doesn't receive now next properties: uploadFileEndpoint, uploadSucceedCallback

    [0.102.*]

    Changed

    • activateFormPackage method now supports two implementations via launchdarkly. Old one when the request is sent to server to perform an operation. New one when fields are updated on client side without any network activity, which should be faster.

    [0.97.*]

    Added

    • New method to Esig SDK - requestAuthToken

    [0.96.*]

    Changed

    • Changed the way CSS resources are loaded for the runtime application. See "Applying custom styles and fonts" section of the "SDK User Guide".

    [0.93.*]

    Changed

    • Renamed method ESIGUI_checkSigningStatus to ESIGUI_getSigningStatus
    • validateCreditCard request now sends additional options taken from fourth parameter: 'ftniRequestOptions': defaultStatus, DoNotUpdateCustomer, DoNotUseCustomerName, DoNotUseCustomerAddress

    [0.92.2]

    Added

    • closeEsigUiApp method introduced in SharedGlobalEventHandlers to handle closing eSigUI application.

    Changed

    • ESIGUI_finishSigning method introduced in ESigEventHandlers to notify host application about closing.

    [0.92.*]

    Added

    • SharedGlobalEventHandlers class. Its instance object is used to extend the GlobalEventHandlers with functionality to start eSigUI:
      • ESIGUI_startSigning
    • initialLoadData method introduced in ESigEventHandlers to handle eSigUI initializing on application run;

    Changed

    • ESigEventHandlers to independent SDK with eSigUI-specific functionality;
    • Extract method ESIGUI_startSigning from ESigEventHandlers to SharedGlobalEventHandlers SDK;

    [0.91.*]

    Added

    • Runtime core API methods: convertSmartyStreetsPredictions, convertGooglePlacesPredictions, parseSmartyStreetsAddress, parseGooglePlacesAddress, getStringProductPropertyByName
    • Global Event Handlers: getSmartyStreetsPredictions, getGooglePlacesPredictions, getSmartyStreetsDetails, getGooglePlacesDetails

    Changed

    • IGo Specific Global Event Handlers' method viewPDF now calls predefined CA if its name was specified.
    • jQuery was updated to 3.3.1
    • bootstrap was updated to 3.3.7 to support 3.x.x versions of jQuery

    [0.90.*]

    Added

    • Runtime core API methods:
      • getDestiny

    [0.89.*]

    Added

    • ESigEventHandlers class. Its instance object is used to extend the GlobalEventHandlers with eSigUI-specific functionality:
      • ESIGUI_acceptConsent
      • ESIGUI_acceptDisclosure
      • ESIGUI_captureEvent
      • ESIGUI_checkSigningStatus
      • ESIGUI_declineConsent
      • ESIGUI_declineDisclosure
      • ESIGUI_finishSigning
      • ESIGUI_flushAuditTrail
      • ESIGUI_getVendorURL
      • ESIGUI_startSigning
      • ESIGUI_viewPDF
    • Runtime core API methods:
      • generateIndexHtmlForNested

    [0.87.*]

    Added

    • Runtime core API methods: renderUploadFileComponent, renderShowAttachmentsComponent, getApiEndpoint
    • Global Event Handlers: uploadFile, showAttachments

    [0.85.*]

    Added

    • Runtime core API methods: getUserDataTypeByName
    • Global Event Handlers: retrieveParamedSchedulerUrl, checkParamedSchedulerStatus

    Changed

    • USER_DATA item is appended to runtime core API object runtimeStateItems
    • Runtime core API method getRuntimeState now supports USER_DATA runtime state item to retrieve the user data object

    [0.84.*]

    Added

    • Runtime core API methods: showModal, hideModal

    Changed

    • (Breaking) exportFormFromIGO from GlobalEventHandlers renamed to viewPDF.

    [0.83.*]

    Added

    • Runtime core API methods: restoreDefaultValues, getRuntimeDirtyData

    [0.82.*]

    Changed

    • (Breaking) Runtime core API method getRuntimeState being called without arguments now returns an object with all runtime state items including previousScreenName. In previous versions of API it returned an object with only previousScreenName)
    • (Warning) Runtime core API methods getRuntimeData and updateRuntimeData are now deprecated and will be removed in one of future releases. getRuntimeState and updateRuntimeState should be used instead
    • Runtime core API method getRuntimeState now accepts array of items to retrieve as a parameter and returns the object with all specified parameters (in previous versions of API it returned an object with only previousScreenName)
    • loadData now accepts data to load as a parameter. If it's undefined then loadData tries to load the data from the server (by default and as it was before).
    • Runtime core API method updateRuntimeData can now also accept data in format returned by getRuntimeData method

    Added

    • Runtime core API methods: resetRuntimeState, restoreRuntimeState, updateRuntimeState, 'getFieldByName'
    • Runtime core API constant: runtimeStateItems which contains names of runtime state items used in getRuntimeState, resetRuntimeState, restoreRuntimeState, updateRuntimeState API methods
    • Runtime core API constant: sdkEventHandlerNames which contains names of events that can be handled via SDK.

    [0.79.*]

    Changed

    The following change is not Breaking unless you use Javascript custom assemblies.

    • Two methods in GlobalEventHandlers: getSourceDropdownData and getSourceGridData are updated to load data from javascript custom assembly first. If JS custom assembly is missing, request is sent to the server.

    [0.77.*]

    Added

    • callJsCustomAssembly method introduced in GlobalEventHandlers to handle new action call_js_service in screen designer

    [0.75.*]

    Changed

    The following changes may be considered as breaking if you are using custom stylesheets instead of default.

    • (Breaking) Controls styles extracted to separate bundle runtime-controls.css. List of changes:

      • Controls styles extracted from runtime.css to runtime-controls.css
      • Styles for radio and checkbox buttons updated and moved from runtime-bootstrap-overrides.css to runtime-controls.css bundle

      • Radio/Checkbox button control's CSS classes updated:

        • btn-group class removed
        • btn-group-vertical class renamed to radio-group-vertical
        • btn and btn-_ classes renamed to input-btn and input-btn-_ classes accordingly

    [0.74.*]

    Changed

    • Third party styles extracted from runtime.css to separate style sheets:
      • runtime-bootstrap-theme.css
      • runtime-bootstrap-overrides.css
      • runtime-fonts.css
      • runtime-font-awesome.css
      • runtime-browser-fixes.css

    [0.74.*]

    Changed

    • validateCreditCard request now sends additional parameters taken from client data: ProcessingInstruction, Field1, Field2, FieldDelimiter, LedgerData

    [0.73.*]

    Added

    • callServerProcedure to GlobalEventHandlers
    • applyEsign to GlobalEventHandlers
    • declineEsign to GlobalEventHandlers
    • activateFormPackage to GlobalEventHandlers
    • exportFormFromIGO to GlobalEventHandlers
    • sendEmail to GlobalEventHandlers
    • callLogger to GlobalEventHandlers

    [0.72.*]

    Added

    • css namespacing page
    • ability to use namespaced css

    [0.71.*]

    2017-10-16

    Changed

    • jQuery and Bootstrap extracted from runtime.js and are loaded separately in scripts.html
    • All scripts in scripts.html are loaded with defer attribute

    [0.70.*]

    2017-09-07

    Added

    • CHANGELOG.md
    • expose runtime core api to global scope (global.App.runtimeCoreApi)
    • add loadData to GlobalEventHandlers
    • add refreshUI method to runtime core api

    Changed

    • (Breaking) rename api to coreApi inside GlobalEventHandlers.js
    • (Breaking) global.GlobalEventHandlers --> global.App.__globalEventHandlers
