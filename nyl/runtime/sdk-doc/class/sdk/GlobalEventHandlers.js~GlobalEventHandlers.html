    public onDynamicTableEvent(tableFieldName: string, type: tableInit | editorClick | editorValueChange | editorFocus | editorBlur | sortChange | pageChange | itemsPerPageChange, args: object): undefined since v0.131.* source

    This method is invoked as a reaction on events happening inside dynamic table control. It's expected that the abstract logic inside would be the next: depending on event type some changes could be made to the control state via specific coreApi method. Then, calling coreApi.refreshUI method, changes to the control state will be applied in UI (control will be re-rendered). See SD Runtime DynamicTable control data in Data Interchange Format description for details relating to the state of DynamicTable field (how it's stored in runtime core and available here via coreApi methods).

    Params:

    NameTypeAttributeDescription
    tableFieldName string

    name of field ( given in property card in designer)
    type tableInit | editorClick | editorValueChange | editorFocus | editorBlur | sortChange | pageChange | itemsPerPageChange

    name of the event
    args object

    specific to event type arguments.
    args.columnName string
    • optional

    name of a column where event is happening
    args.rowId number
    • optional

    index of a row where event is happening
    args.value string | number | boolean
    • optional

    value of an editor passed in editorValueChange event
    args.direction asc | desc
    • optional

    sorting direction for a column
    args.page number
    • optional

    page number for changing page event
    args.itemsPerPageChange number
    • optional

    itemsPerPageChange for changing items per page event

    Return:

    undefined

    Example:

     Args by type:
   - tableInit
         no args
   - editorClick
       - columnName
       - rowId

   - editorValueChange
       - columnName
       - rowId
       - value

   - editorFocus
       - columnName
       - rowId

   - editorBlur
       - columnName
       - rowId

   - sortChange
       - columnName
       - direction

   - pageChange
      - page

   - itemsPerPageChange
      - itemsPerPageChange
    Example of implementation:

ExtensionClass.prototype.onDynamicTableEvent = function(tableFieldName, type, args) {

  // Use only updateFieldStateImmutably coreApi method to update state
  if (type === 'editorValueChange') {
      this.coreApi.updateFieldStateImmutably(fieldName, state => {
         const row = state.value.data.find(row => row.rowId === args.rowId);
         if(!row) return;

         row[args.columnName].value = args.value;
         return state;
      });
  }

  this.coreApi.refreshUI();

}

    public onFieldValidate(field: object, data: object, validations: Validations): Validations | undefined since v0.131.* source

    This method is invoked by runtime core as a hook to internal field validation. Invocation happens right before validation results of different validators (isRequired, maxLength, etc...), which are setup for a specific field, will be merged into final validation status for the field (whether field is valid or not) by the runtime core. This means that here you have full control over validation mechanism of runtime core: you can rewrite result just for one validator in series (for example you'd like to override how isRequired should be calculated for a mask control field) or you can deny all validations made by runtime core and perform your validations here.

    NOTE: Be careful with performance, this method is invoked frequently.

    Params:

    NameTypeAttributeDescription
    field object

    object containing all properties of the field (type, name, label, etc...)
